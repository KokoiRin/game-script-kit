## 背景与动机

游戏自动化脚本很容易把玩法决策和平台相关的鼠标、键盘操作混在一起，导致脚本难以测试、复用和迁移。这个变更会先建立一个小型框架边界，让稳定的业务逻辑留在核心层，把不同平台的设备控制差异隔离到适配器里。

## 变更内容

- 新增脚本核心层，通过抽象输入设备操作表达游戏流程，而不是直接调用操作系统 API。
- 新增输入设备适配器契约，覆盖鼠标移动、点击和拖动。
- 新增一个 macOS Python 初始实现，用来演示鼠标移动、点击和拖动模拟。
- 新增 demo 脚本，验证业务逻辑可以通过抽象接口运行，而不嵌入 macOS 专用代码。
- 不包含破坏性变更；这是一个新的框架和 demo 基础。

## 能力范围

### 新增能力

- `game-script-core`: 定义游戏自动化业务逻辑如何围绕平台无关的输入操作组织。
- `input-device-adapter`: 定义设备控制接口，以及初始 macOS Python 鼠标指针适配器的行为。

### 修改能力

- 无。

## 影响范围

- 新增游戏自动化框架的 Python package 或模块结构。
- 引入设备适配器 API，后续其他平台可以实现同一接口。
- 新增用于鼠标指针模拟的 macOS 相关依赖或集成。
- 新增 demo 入口和聚焦测试，用来验证业务逻辑与平台行为的分离。
